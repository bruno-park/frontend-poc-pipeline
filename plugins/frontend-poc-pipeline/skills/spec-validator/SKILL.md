# 기획서 디스크립션 검증기 (spec-validator)

Figma 화면기획서의 description을 "기획서 디스크립션 정책"에 맞게 검증합니다.
**load**, **validate**, **guide** 세 가지 모드로 동작하며, Confluence 접근 불가 시 로컬 fallback을 자동 사용합니다.

---

## 사용법

```
/spec-validator                    # load: 정책 요약 출력
/spec-validator validate           # validate: 아래에 검증할 디스크립션 입력
/spec-validator guide              # guide: 다른 스킬에서 정책 컨텍스트 주입 시
```

### validate 입력 예시

**2컬럼 형식 (실제 Figma 기준 — 권장):**
```
/spec-validator validate

| no | description                                              |
|----|----------------------------------------------------------|
| 1  | [광고 구성 정보 설정]                                    |
|    | [P] 동영상 광고 > 피드형 선택 시 나오는 화면            |
| 2  | [소재 스펙 구성 항목]                                    |
|    | [P] 동영상 피드형 미디어 스펙 표시                      |
|    | [시간제한 기능]                                          |
|    | [P] 기본 값 : 사용, 5초                                 |
|    | [F] 미사용 선택 시 시간 설정 셀렉트 박스 Disable        |
| 3  | [레이아웃 팝업 오픈]                                     |
|    | [F] 클릭 시 → 'FT-26-05 | MODAL | ...' 페이지 오픈     |
```

**4컬럼 형식 (정책 기준):**
```
/spec-validator validate

| no  | title      | setup              | rules                                      |
|-----|------------|--------------------|--------------------------------------------|
| 1   | 검색 영역  | 기본값: 전체       | [F] 검색 버튼 클릭 시 -> 검색 결과 노출    |
|     |            |                    | [P] 최근 90일 데이터만 조회 가능           |
| 2   | 목록       | 기본 정렬: 등록일  | [Data]                                     |
|     |            |                    | - 광고명: 광고 이름                        |
|     |            |                    | - 등록일: 상품 등록 날짜                   |
```

## 내장 정책 규칙 (Built-in Rules)

> Confluence 조회는 **버전 확인 및 최신 변경 감지** 용도입니다.
> 실제 검증에 사용하는 규칙은 아래에 내장되어 있습니다.

### 구조 규칙

**지원 형식 2가지:**

| 형식 | 컬럼 구성 | 비고 |
|------|-----------|------|
| **2컬럼** (실제 Figma 기준) | `no` + `description` | title을 `[볼드 섹션명]`으로 description 내 포함 |
| **4컬럼** (정책 기준) | `no` + `title` + `setup` + `rules` | setup은 태그 사용 금지 |

| 컬럼 | 설명 | 제약 |
|------|------|------|
| `no` | 화면 번호 태그와 동일 | 숫자 또는 숫자-숫자 형식 (예: 1, 3-1, 6-2) |
| `title` | UI 영역 또는 기능 명칭 (4컬럼 전용) | 명확한 영역명 사용 |
| `setup` | 초기 상태 / 기본 설정 (4컬럼 전용) | **[F][P][Data] 태그 사용 금지**, 없으면 `-` |
| `description` | 섹션 제목 + 태그 기반 규칙 (2컬럼 전용) | `[섹션명]` 볼드 제목 허용; **no가 비어있는 행은 위 번호의 continuation** |
| `rules` | 태그 기반 상세 규칙 (4컬럼 전용) | [F], [P], [Data] 태그만 허용 |

### 태그 규칙

**[섹션 제목] — 대괄호 섹션명 (태그 아님)**
- `[광고 구성 정보 설정]`, `[시간제한 기능]` 등 볼드 처리된 섹션 구분용
- **[F], [P], [Data], [TBD] 외 대괄호 = 섹션 제목으로 허용** (오류 처리 금지)

**[F] — 사용자 액션 / 화면 동작**
- 권장 형식: `[F] '조건' -> '결과'`
- 이동 형식: `[F] {요소} 클릭 시 → '{페이지명}' 페이지로 이동` (탭 옵션: 현재 탭 / 새 탭)
- 자연어 형식도 허용: `[F] 미사용 선택 시 시간 설정 셀렉트 박스 Disable` (경고만)
- 예: `[F] 검색 버튼 클릭 시 -> 결과 목록 갱신`

**[P] — 정책 / 처리 규칙 / 텍스트 / 선택 옵션**
- 예: `[P] 최근 90일 데이터만 조회 가능`
- 예: `[P] 리스트: 전체, ON, OFF`

**[Data] — 테이블 컬럼 정의 전용**
- 블록 구조, 태그 1회만 선언, 하위 필드는 `-` 사용
- 단순 값/레이블/옵션 나열은 [P]로 작성
```
[Data]
- 필드명: 설명
- 필드명: 설명
```

### 작성 순서
`[F] → [P] → [Data]` (흐름상 자연스러우면 일부 변경 허용 — 경고만)

### [validate 모드] 정책 준수 검증

**텍스트 미제공 시:**
```
검증할 디스크립션을 입력해주세요.

입력 형식: 표(table) 또는 행(row) 형태로 no/title/setup/rules를 제공하세요.
자유 텍스트만 제공된 경우 구조 파악이 어려울 수 있습니다.
```

**검증 항목 (10가지):**

| # | 검증 항목 | 기준 | 수준 |
|---|-----------|------|------|
| 1 | 구조 확인 | 2컬럼(no+description) 또는 4컬럼(no+title+setup+rules) 인식 가능 여부 | 오류 |
| 2 | no 형식 | 숫자 또는 숫자-숫자 형식 (1, 3-1) | 경고 |
| 3 | setup 태그 금지 (4컬럼) | setup에 [F], [P], [Data] 미포함 | 오류 |
| 4 | rules/description 빈 값 | 비어있거나 `-`만 있는 경우 | 경고 |
| 5 | 유효 태그만 사용 | [F], [P], [Data], [TBD] 외 대괄호는 섹션 제목으로 허용 | - |
| 6 | [F] 형식 | `'조건' -> '결과'` 또는 `→ '페이지명' 페이지로 이동` 권장 | 경고 |
| 7 | [Data] 블록 구조 | 태그 1회 선언, 하위 필드 `-` 사용 | 오류 |
| 8 | [Data] 컬럼 정의만 | 단순 값/레이블/옵션 나열 금지 | 오류 |
| 9 | 태그 작성 순서 | [F] → [P] → [Data] | 경고 |
| 10 | [TBD] 항목 | 정책 미확정 항목 존재 | 정보 |

**출력 형식 (예시 포함):**
```
--- 기획서 디스크립션 검증 결과 ---

정책 출처: {source}

# 아래는 예시 출력입니다 (실제 결과는 입력에 따라 달라집니다)
[결과] {통과수}개 통과 / {오류수}개 오류 / {경고수}개 경고 / {info수}개 정보

[오류 항목]
| 항목            | 위치         | 위반 내용                          | 수정 제안                              |
|-----------------|--------------|-------------------------------------|----------------------------------------|
| [F] 형식 (#6)   | no=1, rules  | `[F] 클릭시 팝업` → 화살표 형식 없음 | `[F] 버튼 클릭 시 -> 팝업 노출`로 수정 |

[경고 항목]
| 항목              | 내용                                      |
|-------------------|-------------------------------------------|
| 태그 순서 (#9)    | no=2에서 [P] 다음 [F] 작성됨 (허용 범위) |
```

위반 없으면: `모든 항목이 정책을 준수합니다.`

### [guide 모드] PRD/화면기획 작성 시 정책 컨텍스트 주입

다른 스킬(`figma-jira-prd`, `screen-plan` 등)에서 호출:

```
# 다른 스킬에서 guide 모드 호출 방법
/spec-validator guide
→ 반환된 정책 컨텍스트를 PRD 작성 프롬프트 앞에 삽입
```

**출력:**
```
--- 기획서 디스크립션 정책 컨텍스트 ---

이하 규칙을 준수하여 디스크립션을 작성하세요.
출처: {source}

[구조] 2컬럼(no+description) 또는 4컬럼(no+title+setup+rules) 중 하나 선택
[setup] 태그 사용 금지, 없으면 `-`

[F] 사용자 액션 → 화면 동작
    권장: [F] '조건' -> '결과'
    이동: [F] {요소} 클릭 시 → '{페이지명}' 페이지로 이동
    자연어도 허용 (경고): [F] 미사용 선택 시 셀렉트 박스 Disable

[P] 정책, 제약, 텍스트, 기본값, 선택 옵션
    예: [P] 최근 90일 데이터만 조회 가능
    예: [P] 기본 값 : 사용, 5초

[Data] 테이블 컬럼 정의 전용 — 블록 구조, 태그 1회, 하위 `-`
    [Data]
    - 필드명: 설명
    단순 옵션/레이블은 [P]로 작성

[섹션 제목] [볼드 텍스트] 형식으로 구분 — 태그 아님, 허용
    예: [광고 구성 정보 설정], [시간제한 기능]

[순서] [F] → [P] → [Data] (흐름상 자연스러운 경우 변경 허용)
[no형식] 숫자 또는 숫자-숫자 (1, 3-1, 6-2)
[TBD] 정책 미확정 항목: [TBD] 내용 협의 필요
```

## 엣지 케이스

| 상황 | 처리 방법 |
|------|-----------|
| Confluence 연결 실패 | 내장 규칙으로 진행 + 경고 표시 |
| fallback 파일도 없음 | 에러 메시지 + 양쪽 경로 안내 후 종료 |
| 문서 버전 변경 감지 | Confluence 버전이 fallback(v17)과 다를 경우 알림 |
| validate 텍스트 미제공 | 입력 형식 안내 후 종료 |
| 자유 텍스트로 입력된 경우 | "표 형식으로 다시 입력해주세요" 안내, 구조 파악 시도 후 실패 시 종료 |
| no가 숫자 형식이 아닌 경우 | 경고 — 정책상 숫자 기반 태그와 매칭되어야 함 |
| rules가 비어있는 경우 | 경고 — 빈 rules는 허용되나 확인 권장 |
| 태그 순서 혼합 | 경고만 (흐름상 자연스러운 경우 허용) |
| [Data] 단순 값 나열 | 오류 — [P]로 이동 제안 |
| setup에 태그 포함 | 오류 — 텍스트만 허용 |
| no 세부 번호 (3-1) | 정상 — 세부 번호 체계 허용 |
